Which JS approach for ACE?

You want: minimalist code, minimalist UI, no fighting CSS, easy debugging

A

Vanilla JS + HTMX

No build step. No npm. No Node.js.

FastAPI + Jinja2 templates + HTMX for partial page updates + plain JS for interactions.

Pros

  • Zero dependencies beyond HTMX (~14 KB) + SortableJS (~13 KB)
  • What you write is what runs — no compilation, no abstraction
  • Debug with browser DevTools directly, no source maps needed
  • Your existing bridge.js transfers almost unchanged
  • Simplest possible mental model

Cons

  • More boilerplate for toggles, dropdowns, conditional visibility (~10-15 lines each vs 3 with Alpine)
  • No reactive data binding — manual DOM updates
  • Keyboard shortcut wiring is more verbose

Total JS overhead: ~27 KB gzipped • Files to learn: HTMX docs only

B

Alpine.js + HTMX

No build step. Declarative reactivity via HTML attributes.

Everything in Option A, plus Alpine.js for toggles, dropdowns, keyboard shortcuts, and reactive show/hide.

Pros

  • Toggles/dropdowns in 3 lines instead of 15 (x-show, x-on)
  • Keyboard shortcuts are clean: @keydown.ctrl.z.window="undo()"
  • Still no build step, no npm, no Node.js
  • Large community in Python/Django/FastAPI ecosystem

Cons

  • HTMX DOM swaps destroy Alpine state — requires morph extension to fix
  • Alpine's MutationObserver slows DOM-heavy pages (annotation spans, drag-and-drop)
  • Debugging state-loss between HTMX + Alpine is notoriously hard
  • Doesn't help with the hard parts (text selection, DnD, undo/redo)

Total JS overhead: ~44 KB gzipped • Files to learn: HTMX + Alpine docs + morph extension

C

Svelte 5 (or Preact)

Build step required. Full reactive component model.

Separate Svelte frontend compiled by Vite, FastAPI as pure API backend.

Pros

  • Best DX for complex reactive UI (shared state across panels)
  • Cleanest drag-and-drop via svelte-dnd-action
  • Smallest compiled output (~2-3 KB runtime)
  • Excellent devtools and TypeScript support

Cons

  • Requires Node.js, npm, Vite, config files
  • Two dev servers during development (Vite + FastAPI)
  • Two languages to maintain (Python + Svelte/JS)
  • CORS/proxy setup between frontend and backend
  • Overkill for ACE's level of interactivity

Total JS overhead: ~25-35 KB gzipped • Files to learn: Svelte + Vite + SvelteKit docs

My recommendation: A

Your hard problems (text selection offsets, drag-and-drop, undo/redo) are all vanilla JS regardless of framework. Alpine saves ~50-100 lines of toggle boilerplate but introduces HTMX state-loss headaches. Svelte is great but adds a whole build toolchain for an app that doesn't need it.

Vanilla JS + HTMX + SortableJS = fewest moving parts, easiest to debug, your existing bridge.js carries over, and you write CSS directly in files you control.
